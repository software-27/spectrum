## Overview

A **TabList** creates a set of **Tab** elements. This is typically used with the **TabView** component to control a set of set of layered sections of content, known as tab panels, that display one panel of content at a time.

## Example

<TabList>
  <Tab>
    Tab 1
  </Tab>
  <Tab>
    Tab 2
  </Tab>
  <Tab>
    Tab 3
  </Tab>
</TabList>

```jsx
<TabList>
  <Tab>
    Tab 1
  </Tab>
  <Tab>
    Tab 2
  </Tab>
  <Tab>
    Tab 3
  </Tab>
</TabList>
```

### autoFocus

Use `autoFocus` to automatically set focus to the selected Tab when the TabList appears in a Dialog or Popover.

```jsx
<TabList autoFocus>
  <Tab>Tab 1</Tab>
  <Tab selected>Tab 2</Tab>
  <Tab>Tab 3</Tab>
</TabList>
```

## Accessibility

### WAI-ARIA Design Pattern

The TabList implements the Tab List portion of the Tabs design pattern, per [WAI-ARIA 1.1](https://www.w3.org/TR/wai-aria-practices-1.1/#tabpanel).
See the [TabView](/components/TabView) component, for a complete implementation of the design pattern that includes tab panel content.

### Keyboard activation

By default, a Tab automatically becomes selected when it receives focus. By setting the `keyboardActivation` prop to `"manual"`, a user will need to press the <kbd>Enter</kbd> or <kbd>Space</kbd> key after a Tab has received focus to select.

#### Automatic (default)
<TabList keyboardActivation="automatic">
  <Tab>Tab 1</Tab>
  <Tab>Tab 2</Tab>
</TabList>

```jsx
<TabList keyboardActivation="automatic">
  <Tab>Tab 1</Tab>
  <Tab>Tab 2</Tab>
</TabList>
```

#### Manual
<TabList keyboardActivation="manual">
  <Tab>Tab 1</Tab>
  <Tab>Tab 2</Tab>
</TabList>

```jsx
<TabList keyboardActivation="manual">
  <Tab>Tab 1</Tab>
  <Tab>Tab 2</Tab>
</TabList>
```

### Labelling with `aria-label` or `aria-labelledby`

A TabList can be labelled, using either the `aria-label` or `aria-labelledby` attribute, to provide additional context for Accessibility.

  <Tab icon={<Instagram alt="Instagram" />} title="Instagram" />
</TabList>
