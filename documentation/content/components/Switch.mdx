## Overview

A **Switch** is used to turn an option on or off. Switches allow users to select the state of a single option at a time. Use a switch rather than a checkbox when activating (or deactivating) an option, instead of selecting.

## Example

<Switch label="Switch" onChange={e => console.log(e)} />

```jsx
<Switch label="Switch" onChange={e => console.log(e)} />
```

## Variants

### Standard Switches

Standard switches are the default style for switches. The blue color provides a visual prominence that is optimal for forms, settings, etc. where the switches need to be noticed.

<Switch label="React" defaultChecked />

```jsx
<Switch label="React" defaultChecked />
```

### Quiet Switches

Quiet switches are a secondary style for switches. The gray color provides a less prominent style than the standard switches. They are optimal for application panels where all visual elements are monochrome in order to direct focus to the content.

<Switch label="React" defaultChecked quiet />

```jsx
<Switch label="React" defaultChecked quiet />
```

## Accessibility

Switch are accessible by default, rendered in HTML using the `<input type="checkbox">` element with the appropriate accessibility role, `switch`. When the Switch is `checked` or `invalid`, the appropriate ARIA state attribute will automatically be applied.

### Labelling

Switch must have a label to be properly identified to users of assistive technology. The easiest way to satisfy this requirement is by adding a `label` prop or by including the label text as child content.

<div>`label` prop: </div>
<Switch label="React" />

<div>as content: </div>
<Switch defaultChecked>React</Switch>

<div>not the best practice: </div>
<Switch renderLabel={false} label="React switch" defaultChecked>with renderLabel: false</Switch>

```jsx
{/* `label` prop */}
<Switch label="React" />

{/* as content */}
<Switch defaultChecked>React</Switch>

{/* not the best practice: */}
<Switch renderLabel={false} label="React switch" defaultChecked>with renderLabel: false</Switch>
```

There are occassionally use cases that require the Switch not have a visible label, for example in a TableView to show row selection. In these cases, an accessible label can be added using `aria-label` or `aria-labelledby`, or the `label` prop can be hidden by setting `renderLabel` to `false`.

<Table>
  <THead>
    <TH scope="column">
      Active
    </TH>
    <TH scope="column">
      Name
    </TH>
  </THead>
  <TBody>
    <TR>
      <TD style={{'padding-top': '8px'}}>
        {/* Using both aria-label and aria-labelledby with self-reference: */}
        <Switch
          id="row0-switch"
          aria-label="Active"
          aria-labelledby="row0-switch row0-cell1" />
      </TD>
      <TD id="row0-cell1" role="rowheader" scope="row">
        Filename 1
      </TD>
    </TR>
    <TR>
      <TD style={{'padding-top': '8px'}}>
        {/* Using renderLabel={false} and aria-labelledby with self-reference: */}
        <Switch
          id="row1-switch"
          aria-labelledby="row1-switch row1-cell1"
          renderLabel={false}>
            Active
        </Switch>
      </TD>
      <TD id="row1-cell1" role="rowheader" scope="row">
        Filename 2
      </TD>
    </TR>
  </TBody>
</Table>
