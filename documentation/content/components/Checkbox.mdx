## Overview

**Checkboxes**  allow users to select multiple items from a list of independent options, or to mark an individual option as selected.

Should I use a checkbox or a switch? Use a switch when activating something instead of selecting.

### Example

<Checkbox label="React"/>

```jsx
<Checkbox label="React"/>
```

### Standard Checkboxes
Standard checkboxes are the default style for checkboxes. The blue color provides a visual prominence that is optimal for forms, settings, lists or grids of assets, etc. where the checkboxes need to be noticed.


<Checkbox label="React" defaultChecked />

```jsx
<Checkbox label="React" defaultChecked />
```

### Quiet Checkboxes

Quiet checkboxes are a secondary style for checkboxes. The gray color provides a less prominent style than the standard checkboxes. They are optimal for application panels where all visual elements are monochrome in order to direct focus to the content.

<Checkbox label="React" defaultChecked quiet />

```jsx
<Checkbox label="React" defaultChecked quiet />
```

### States

In addition to the variant, Checkboxes have a number of props for controlling their visual state. All checkbox variants support the `disabled`, `indeterminate`, `invalid`   prop, which applies a disabled style to the checkbox, and also prevents clicks from activating it.

<div>checked: </div><Checkbox label="React" checked/>

<div>indeterminate: </div><Checkbox label="React" indeterminate/>

<div>invalid: </div><Checkbox label="React" invalid/>

<div>disabled: </div><Checkbox label="React" disabled/>

```jsx

<div>defaultChecked: </div><Checkbox label="React" defaultChecked/>
```

### Handling Events

Event handlers for clicks and other user actions can be registered on a `Checkbox` just as a normal `Checkbox` element in React.

<Checkbox onChange={e => console.log(e)}>Click me!</Checkbox>

```jsx
<Checkbox onChange={e => console.log(e)}>Click me!</Checkbox>
```

## Internationalization

To internationalize a checkbox, a localized string should be passed to the `label` prop of the checkbox, or as a child.

## Accessibility

Checkboxes are accessible by default, rendered in HTML using the `<input type="checkbox">` element. When the checkbox is set as `indeterminate` or `invalid`, the appropriate ARIA state attribute will automatically be applied.

### Labelling

Checkbox must have a label to be properly identified to users of assistive technology. The easiest way to satisfy this requirement is by adding a `label` prop or by including the label text as child content.

<div><code>label</code> prop: </div><Checkbox label="React" />

<div>as content: </div><Checkbox defaultChecked>React</Checkbox>

<div>not the best practice: </div><Checkbox renderLabel={false} label="React checkbox" defaultChecked>with renderLabel: false</Checkbox>

```jsx
<div><code>label</code> prop: </div><Checkbox label="React" />

<div>as content: </div><Checkbox defaultChecked>React</Checkbox>

<div>not the best practice: </div><Checkbox renderLabel={false} label="React checkbox" defaultChecked>with renderLabel: false</Checkbox>
```

There are occassionally use cases that require the Checkbox not have a visible label, for example in a TableView to show row selection. In these cases, an accessible label can be added using `aria-label` or `aria-labelledby`, or the `label` prop can be hidden by setting `renderLabel` to `false`.
