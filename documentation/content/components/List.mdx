## Overview

Lists are used to group together related pieces of information so they are clearly associated with each other, easy to read, and navigable using the keyboard. The various items in the list are provided using the `<ListItem/>` component, which can have a icon, label or both. List is the base component for the [SelectList](/components/SelectList) and [Menu](/components/Menu) components.

## States

ListItems have a couple of props for controlling their visual state. The `disabled` prop applies a disabled style to the listItem, and also prevents clicks from activating it. The `selected` prop applies a selected appearance to the listItem.

### Selected

<List aria-label="List">
  <ListItem selected>
    Hello
  </ListItem>
  <ListItem>
    World
  </ListItem>
</List>

```jsx
<List aria-label="List">
  <ListItem selected>
    Hello
  </ListItem>
  <ListItem>
    World
  </ListItem>
</List>
```

## Handling Events

Events handlers for select can be registered on a `ListItem` as shown below.

<List aria-label="List">
  <ListItem value="Hello" onSelect={(value, event) => console.log('onSelect', event.type, value)}>
    Hello
  </ListItem>
  <ListItem value="World">
    World
  </ListItem>
</List>

```jsx
<List aria-label="List">

    Hello
  </ListItem>
  <ListItem value="World">
    World
  </ListItem>
</List>
```

## Internationalization

To internationalize a ListItem, a localized string should be passed as a child.

## Accessibility

### WAI-ARIA Design Pattern

The List component implements the [Listbox design pattern, per WAI-ARIA 1.1](https://www.w3.org/TR/wai-aria-practices-1.1/#Listbox), for static role and state properties, and keyboard navigation.
The List does not provide for dynamic item selection when `onSelect` is called on a listItem. The `selected` state is a controlled prop on ListItem.
The [SelectList](/components/SelectList), [Select](/components/Select), and [Menu](/components/Menu) components implement dynamic item selection, and are more likely to be used as standalone components.

### Labelling

A List should be labelled, using either the `aria-label` or `aria-labelledby` attribute, to provide context for Accessibility.
