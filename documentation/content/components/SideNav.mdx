## Overview

Side navigation allows users to locate information and features within the UI. It can be used for either hierarchical or flat navigation, and gives the ability to group navigable items categorically. Side navigation is an opportunity to prioritize content or features based on your users’ needs in a way that maintains clear, persistent visibility. Use side navigation within the context of larger elements and mechanisms within the app frame.

## autoFocus: true

When `autoFocus` is set to `true`, focus will be set to the selected item when the component mounts. This is useful to focus the SideNav on url changes or within a dialog and works best when `manageTabIndex` is also `true`.

⇐ The `autoFocus` behavior is demonstrated by the side navigation for this React Spectrum documentation site.

  <SideNav value="Docs" autoFocus manageTabIndex typeToSelect>

## Controlled

Use `value` prop to set controlled value. With a `value` prop, ancestor component should handle to `onSelect` event to update `value` with a state variable.

  <ControlledSideNav value="Community" manageTabIndex typeToSelect onSelect={value => console.log('New value: "' + value + '" selected')}>
    <SideNavItem value="Docs">Docs</SideNavItem>
    <SideNavItem value="Guides">Guides</SideNavItem>
    <SideNavItem value="Community">Community</SideNavItem>
    <SideNavItem value="Storybook">Storybook</SideNavItem>
    <SideNavItem value="Releases" disabled>Releases</SideNavItem>
    <SideNavItem value="GitHub">GitHub</SideNavItem>
  </ControlledSideNav>

```jsx
  import Provider from '@react/react-spectrum/Provider';
  import React from 'react';
  import ReactDOM from 'react-dom';
  import {SideNav} from '@react/react-spectrum/SideNav';

  class ControlledSideNav extends React.Component {
    state = {
      value: this.props.defaultValue || this.props.value
    }
    onSelect(value, e) {
      if (this.props.onSelect) {
        this.props.onSelect(value, e);
      }
      this.setState({value});
    }
    render() {
      const {
        children,
        ...otherProps
      } = this.props;
      return <SideNav
               {...otherProps}
               value={this.state.value}
               onSelect={this.onSelect.bind(this)}>
        {children}
      </SideNav>;
    }
  }

  let app = (
    <Provider theme="dark">
      <ControlledSideNav
        value="Community"
        manageTabIndex
        typeToSelect
        onSelect={value => console.log('New value: "' + value + '" selected')}>
        <SideNavItem value="Docs">Docs</SideNavItem>
        <SideNavItem value="Guides">Guides</SideNavItem>
        <SideNavItem value="Community">Community</SideNavItem>
        <SideNavItem value="Storybook">Storybook</SideNavItem>
        <SideNavItem value="Releases" disabled>Releases</SideNavItem>
        <SideNavItem value="GitHub">GitHub</SideNavItem>
      </ControlledSideNav>
    </Provider>
  );

  ReactDOM.render(app, dom);
```

## Multi-Level

Use this variation when you have multiple layers of hierarchical navigation. The headers are styled differently and possess the same interactive behavior as a treeview; clicking the header opens and collapses the children navigation items. In the instances where a top-level navigation item has no children, clicking will send the user to the location of the item. Each level can be either “visited” or “expanded”, but never both.

## Accessibility

Both SideNav variants support navigation between items using the <kbd>UP</kbd>/<kbd>DOWN</kbd> arrow keys, <kbd>HOME</kbd> or <kbd>END</kbd>. When `manageTabIndex` is `false`, all items will also receive keyboard focus using the <kbd>TAB</kbd> key.

The SideNav component implements screen reader accessibility somewhat differently depending on the variant.

### Labelling the navigation region

Label the SideNav navigation landmark region using `aria-label` or `aria-labelledby` prop.

A single level SideNav is a navigation landmark region. Items within it are structured as a list of links.
