## Overview

A **TabView** creates a set of **Tab** elements which control a set of set of layered sections of content, known as tab panels, that display one panel of content at a time.

## Example

<TabView>
  <Tab label="Tab 1">
    Tab Body 1
  </Tab>
  <Tab label="Tab 2">
    Tab Body 2
  </Tab>
</TabView>


```jsx
<TabView>
  <Tab label="Tab 1">
    Tab Body 1
  </Tab>
  <Tab label="Tab 2">
    Tab Body 2
  </Tab>
</TabView>
```

### autoFocus

Use `autoFocus` to automatically set focus to the selected Tab when the TabView appears in a Dialog or Popover.

```jsx
<TabView autoFocus>
  <Tab label="Tab 1">
    Tab Body 1
  </Tab>
  <Tab label="Tab 2" selected>
    Tab Body 2
  </Tab>
  <Tab label="Tab 3">
    Tab Body 3
  </Tab>
</TabView>
```

## Accessibility

### WAI-ARIA Design Pattern

The TabView implements the Tabs design pattern, per [WAI-ARIA 1.1](https://www.w3.org/TR/wai-aria-practices-1.1/#tabpanel).

### Keyboard activation

By default, a Tab automatically becomes selected when it receives focus. By setting the `keyboardActivation` prop to `"manual"`, a user will need to press the <kbd>Enter</kbd> or <kbd>Space</kbd> key after a Tab has received focus to select.

#### Automatic (default)
<TabView keyboardActivation="automatic">
  <Tab label="Tab 1">
    Tab Body 1
  </Tab>
  <Tab label="Tab 2">
    Tab Body 2
  </Tab>
</TabView>

```jsx
<TabView keyboardActivation="automatic">
  <Tab label="Tab 1">
    Tab Body 1
  </Tab>
  <Tab label="Tab 2">
    Tab Body 2
  </Tab>
</TabView>
```

#### Manual
<TabView keyboardActivation="manual">
  <Tab label="Tab 1">
    Tab Body 1
  </Tab>
  <Tab label="Tab 2">
    Tab Body 2
  </Tab>
</TabView>

```jsx
<TabView keyboardActivation="manual">
  <Tab label="Tab 1">
    Tab Body 1
  </Tab>
  <Tab label="Tab 2">
    Tab Body 2
  </Tab>
</TabView>
```

### Labelling with `aria-label` or `aria-labelledby`
