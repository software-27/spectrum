## Overview

A **Radio Group** allow users to select only one single option from a list of mutually exclusive options. Unlike dropdowns, radio groups expose all possible choices up front for comparison. It is mandatory to import Radio before RadioGroup. Also each Radio Button must have a value property.

## Example

<RadioGroup name="radio-group">
  <Radio label="1" value="1" />
  <Radio label="2" value="2" />
  <Radio label="3" value="3" />
</RadioGroup>

```jsx
<RadioGroup name="radio-group">
  <Radio label="1" value="1" />
  <Radio label="2" value="2" />
  <Radio label="3" value="3" />
</RadioGroup>
```

## Variants

RadioGroup offers a couple of variants in terms of buttons alignment and the position of labels across the buttons.

### Alignment

The various Radio buttons of the group can either be horizontally (default) or vertically aligned. The latter can be achieved via the 'vertical' prop.

#### Horizontal (Default)

<RadioGroup name="horizontal-group">
  <Radio label="1" value="1" />
  <Radio label="2" value="2" />
  <Radio label="3" value="3" />
</RadioGroup>

```jsx
<RadioGroup name="horizontal-group">
  <Radio label="1" value="1" />
  <Radio label="2" value="2" />
  <Radio label="3" value="3" />
</RadioGroup>
```
#### Vertical

<RadioGroup name="vertical-group" vertical>
  <Radio label="1" value="1" />
  <Radio label="2" value="2" />
  <Radio label="3" value="3" />
</RadioGroup>

```jsx
<RadioGroup name="vertical-group" vertical>
  <Radio label="1" value="1" />
  <Radio label="2" value="2" />
  <Radio label="3" value="3" />
</RadioGroup>
```

### Placement of Labels

The button labels can either be adjacent (default) or below the buttons. The latter can be achieved via the 'labelsBelow' prop.

#### Labels Adjacent

## Handling Events

Events handlers for change can be registered on a `RadioGroup` as shown below.

```jsx
<RadioGroup name="group" onChange={e => console.log(e)}>
  <Radio label="1" value="1" />
  <Radio label="2" value="2" />
  <Radio label="3" value="3" />
</RadioGroup>
```

## Internationalization

To internationalize a radio button, a localized string should be passed to the `label` prop of the radio button.

## Accessibility

Radios are accessible by default, rendered in HTML using the `<input type="radio">` element. When the radio is `checked`, the appropriate ARIA state attribute will automatically be applied.

### Labelling

A Radio must have a label to be properly identified to users of assistive technology. The easiest way to satisfy this requirement is by adding a `label` prop or by including the label text as child content.

<div><code>label</code> prop: </div><Radio label="React" />

<div>as content: </div><Radio defaultChecked>React</Radio>

<div>not the best practice: </div><Radio renderLabel={false} label="React radio" defaultChecked>with renderLabel: false</Radio>

```jsx
<div><code>label</code> prop: </div><Radio label="React" />

<div>as content: </div><Radio defaultChecked>React</Radio>

<div>not the best practice: </div><Radio renderLabel={false} label="React radio" defaultChecked>with renderLabel: false</Radio>
```

#### RadioGroup

`RadioGroup` should be used to label a group of related `Radio` as a fieldset. The RadioGroup label can be provided using `aria-label` or `aria-labelledby`.

<FieldLabel label="Love me some React!" id="radiogroup-label" />
<RadioGroup name="radiogroup" aria-labelledby="radiogroup-label" vertical>
  <Radio label="Yes" value="Yes" />
  <Radio label="No" value="No" />
</RadioGroup>

```jsx
<FieldLabel label="Love me some React!" id="radiogroup-label" />
<RadioGroup name="radiogroup" aria-labelledby="radiogroup-label" vertical>
  <Radio label="Yes" value="Yes" />
  <Radio label="No" value="No" />
</RadioGroup>
```
