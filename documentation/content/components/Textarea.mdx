import FieldLabel from '@react/react-spectrum/FieldLabel';
import Textarea from '@react/react-spectrum/Textarea';

## Overview

The **Textarea** provides an input element for users to enter in data (for forms and such).

## Examples

<Textarea placeholder="Text here" aria-label="Textarea" />

```jsx
<Textarea placeholder="Text here" />
```

## Variants

### Quiet

<Textarea quiet placeholder="Quiet variant" aria-label="Quiet variant" />

```jsx
<Textarea quiet placeholder="Quiet variant" />
```

### Disabled

Disables textarea from interaction

<Textarea disabled placeholder="Disabled variant" aria-label="Disabled variant" />

```jsx
<Textarea disabled placeholder="Disabled variant" />
```

### Validation State

There are two validation states: valid, and invalid.

<Textarea validationState="valid" placeholder="Valid" style={{marginRight: 20}} aria-label="Valid state" />
<Textarea validationState="invalid" placeholder="Invalid" aria-label="Invalid state" />

```jsx
<Textarea validationState="valid" placeholder="Valid" />
<Textarea validationState="invalid" placeholder="Invalid" />
```

### Read-only

Allows text to be selected but not changed

<Textarea readOnly value="Read-only variant" aria-label="Read-only variant" />

```jsx
<Textarea readOnly value="Read-only variant" />
```

### Required

Shows warning if textarea is empty

<Textarea required placeholder="Required variant" aria-label="Required variant" />

```jsx
<Textarea required placeholder="Required variant" />
```

## Events

### onChange

Function fired when value of textarea is changed.

<Textarea onChange={e => console.log(e)} aria-label="Textarea with onChange event handler" />

```jsx
<Textarea onChange={e => console.log(e)} />
```

### onFocus

Function fired when the textarea is focused.

<Textarea onFocus={e => console.log(e)} aria-label="Textarea with onFocus event handler" />

```jsx
<Textarea onFocus={e => console.log(e)} />
```

### onBlur

Function fired when the focus on the textarea is lost.

<Textarea onBlur={e => console.log(e)} aria-label="Textarea with onBlur event handler" />

```jsx
<Textarea onBlur={e => console.log(e)} />
```

## Additional Behavior

### autoFocus

Controls whether or not the textarea will automatically receive focus when it mounts.

```jsx
<Textarea autoFocus />
```

## Accessibility

Textarea has been implemented using the [HTML5 `textarea` control](https://www.w3.org/TR/html52/sec-forms.html#the-textarea-element), and provided it is labelled appropriately, is be accessible by default. 

### Labelling

A Textarea must be labelled using a [FieldLabel](/components/FieldLabel), `aria-label` or `aria-labelledby`, to provide context for Accessibility.

#### Implicit labelling with `FieldLabel`

<FieldLabel label="Comment">
  <Textarea />
</FieldLabel>

```jsx
<FieldLabel label="Comment">
  <Textarea />
</FieldLabel>
```

#### Explicit labelling using `FieldLabel` and `labelFor` property

A FieldLabel can explicitly label a Textarea using the `labelFor` prop.

<FieldLabel label="Comment" labelFor="textarea-id" />
<Textarea id="textarea-id" />

```jsx
<FieldLabel label="Comment" labelFor="textarea-id" />
<Textarea id="textarea-id" />
```

#### Using `aria-label` and `aria-labelledby`
Use `aria-label` and `aria-labelledby` in rare cases, like the [Clock](Clock) component, where the TextField does not have a visible label or where the Textfield should be labelled by an external element.

<div id="post-0-id">
  <p>Social media post content</p>
</div>
<Textarea
  id="post-0-reply-id"
  aria-label="Reply regarding: "
  aria-labelledby="post-0-reply-id post-0-id"
  placeholder="Write your reply…" />

```jsx
<div id="post-0-id">
  <p>Social media post content</p>
</div>
<Textarea
  id="post-0-reply-id"
  aria-label="Reply regarding: "
  aria-labelledby="post-0-reply-id post-0-id"
  placeholder="Write your reply…" />
```
