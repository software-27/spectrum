## renderItem

You can use the `renderItem` prop to customize the rendering of the items displayed in the dropdown.
This is useful in case custom props are needed for styling or accessibility, or using a semantic
element for the type of data being displayed (e.g. `<time>`).

```jsx
<SelectList
  {/* ... */}
  renderItem={item => <time {/* ... */}>{item.label}</time>} />
```

## Accessibility

### WAI-ARIA Design Pattern

The SelectList control implements the [Listbox design pattern per WAI-ARIA 1.1](https://www.w3.org/TR/wai-aria-practices-1.1/#Listbox).

### Labelling

A SelectList should be labelled using a [FieldLabel](/components/FieldLabel), `aria-label` or `aria-labelledby`, to provide context for Accessibility.

#### Using `aria-label`

```jsx
<SelectList
  aria-label="Ice cream flavor"
  placeholder="Select a flavor..."
  options={[{label: "Chocolate", value: "chocolate"}, {label: "Vanilla", value: "vanilla"},
           {label: "Strawberry", value: "strawberry"}, {label: "Caramel", value: "caramel"},
           {label: "Cookies and Cream", value: "cookiesandcream"}, {label: "Peppermint", value: "peppermint"},
           {label: "Some crazy long value that should be cut off", value: "longval"}]}
/>
```

#### Using FieldLabel

When labelling using a FieldLabel, use implicit labelling or add the `aria-labelledby` prop to explicitly associate the SelectList control to the FieldLabel.

##### Implicit labelling will add `aria-labelledby` automatically

<FieldLabel label="Ice cream flavor">
<SelectList
  placeholder="Select a flavor..."
  options={[{label: "Chocolate", value: "chocolate"}, {label: "Vanilla", value: "vanilla"},
           {label: "Strawberry", value: "strawberry"}, {label: "Caramel", value: "caramel"},
           {label: "Cookies and Cream", value: "cookiesandcream"}, {label: "Peppermint", value: "peppermint"},
           {label: "Some crazy long value that should be cut off", value: "longval"}]}
/>
</FieldLabel>
